# 多融合算法结果数据集转换工具使用说明

## 概述

本工具支持将**多个独立的融合算法结果**批量转换为YOLO格式数据集。每个融合算法可以配置一个或多个图像目录，所有算法共享统一的处理流程。

---

## 配置文件说明 (config.py)

### FUSION_DATASETS 配置结构

```python
FUSION_DATASETS = {
    '数据集键名': {
        'name': '数据集显示名称',
        'image_dirs': [          # 支持一个或多个图像目录
            '路径/到/图像目录1',
            '路径/到/图像目录2',  # 同一算法的多个结果目录
        ],
        'label_dir': '路径/到/标签目录',  # 对应标签目录
        'output_dir': '输出/数据集目录',
        'yaml_path': '输出/配置文件.yaml',
        'nc': 6,                  # 类别数量
        'names': ['类别1', '类别2', ...]  # 类别名称列表
    },
    # 可继续添加更多数据集...
}
```

### 当前已配置的数据集

| 配置键名 | 显示名称 | 图像目录 | 标签目录 | 输出目录 |
|---------|---------|---------|---------|---------|
| `WT_dataset` | WT_dataset | ir (1个目录) | labels | data/ir_yolo_dataset |
| `LP_dataset` | LP_dataset | vi (1个目录) | labels | data/vi_yolo_dataset |

## 使用方法

### 1. 查看可用数据集
```bash
python preprocess.py
```
输出示例：
```
用法: python preprocess.py <数据集键名>
可用数据集: ['WT_dataset', 'LP_dataset']

示例:
  python preprocess.py WT_dataset    # 处理第一个数据集
  python preprocess.py LP_dataset    # 处理第二个数据集
  python preprocess.py all           # 处理所有数据集
```

### 2. 处理单个数据集
```bash
# 处理WT数据集
python preprocess.py WT_dataset

# 处理LP数据集
python preprocess.py LP_dataset
```

### 3. 批量处理所有数据集
```bash
python preprocess.py all
```

### 4. 通过主程序运行
```bash
# 预处理指定数据集
python main.py --mode preprocess --dataset WT_dataset

# 运行完整流程（预处理+训练+评估）
python main.py --mode full --dataset WT_dataset
```

## 如何添加新的融合算法结果？

### 情况A：添加一个新的融合算法结果
```python
# 在 config.py 的 FUSION_DATASETS 中添加：
'fusion_algo_1': {
    'name': 'my_fusion_algorithm_1',
    'image_dirs': [
        'E:/path/to/my/fusion/result',  # 融合结果图像目录
    ],
    'label_dir': 'E:/path/to/labels',   # 对应标签目录
    'output_dir': 'data/fusion_algo_1',
    'yaml_path': 'data/fusion_algo_1.yaml',
    'nc': 6,
    'names': ['Bus', 'Car', 'Lamp', 'Motorcycle', 'People', 'Truck']
},
```

### 情况B：同一算法有多个结果目录
```python
'fusion_algo_2': {
    'name': 'my_fusion_algorithm_2',
    'image_dirs': [
        'E:/path/to/fusion2/result_part1',  # 第一部分结果
        'E:/path/to/fusion2/result_part2',  # 第二部分结果
        'E:/path/to/fusion2/result_part3',  # 可继续添加...
    ],
    'label_dir': 'E:/path/to/labels',
    'output_dir': 'data/fusion_algo_2',
    'yaml_path': 'data/fusion_algo_2.yaml',
    'nc': 6,
    'names': ['Bus', 'Car', 'Lamp', 'Motorcycle', 'People', 'Truck']
},
```

## 输出目录结构

每个融合算法的处理结果独立保存，结构如下：

```
data/
├── ir_yolo_dataset/        # WT数据集输出
│   ├── images/
│   │   ├── train/    # 训练集图像 (80%)
│   │   └── val/      # 验证集图像 (20%)
│   └── labels/
│       ├── train/    # 训练集标签
│       └── val/      # 验证集标签
├── ir_data.yaml      # WT数据集配置
│
├── vi_yolo_dataset/        # LP数据集输出
│   ├── images/
│   │   ├── train/
│   │   └── val/
│   └── labels/
│       ├── train/
│       └── val/
└── vi_data.yaml      # LP数据集配置
```

# YOLOv5 数据集配置

## 训练与评估

```bash
# 训练指定融合算法数据集
python train.py WT_dataset
python train.py LP_dataset

# 评估模型（训练完成后）
python evaluate.py WT_dataset
python evaluate.py LP_dataset

# 批量训练所有数据集
for ds in WT_dataset LP_dataset; do python train.py $ds; done
```

## 注意事项

1. **标签格式要求**：标签文件必须为YOLO格式（归一化的中心坐标+宽高+类别），文件名与图像文件对应（`image1.png` → `image1.txt`）

2. **多目录合并**：同一数据集配置多个 `image_dirs` 时，会自动合并所有目录中的有效图像（存在对应标签则保留）

3. **独立输出**：每个融合算法的处理结果独立存放，便于后续对比实验

4. **路径配置**：支持绝对路径和相对路径，路径分隔符使用 `/` 或 `\` 均可

5. **数据过滤**：自动过滤无对应标签的图像，确保数据集完整性

6. **划分比例**：当前配置为训练集80%、验证集20%（无测试集），可在 `DATA_SPLIT` 中调整

## 故障排除

**问题1：提示"警告：在以下目录中未找到带对应标签的有效图像"**
- 检查 `image_dirs` 和 `label_dir` 路径是否正确
- 确认图像文件（.png/.jpg/.jpeg）与标签文件（.txt）的 basename 是否一致
- 检查标签文件是否存在于 `label_dir` 目录下

**问题2：提示"未知数据集: xxx"**
- 运行 `python preprocess.py` 查看可用的数据集键名
- 检查输入的键名是否与 `FUSION_DATASETS` 中的键完全一致（区分大小写）

**问题3：权限错误**
- 确保输出目录所在分区有足够空间
- 检查输出目录是否有写入权限
- 或修改 `config.py` 中的 `output_dir` 到其他位置

**问题4：训练时找不到数据集**
- 检查生成的 `.yaml` 文件中的 `path` 路径是否正确
- 确保预处理已完成，输出目录结构完整
